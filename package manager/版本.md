# 版本的格式

### **major.minor.patch**

主版本号.次版本号.修补版本号

**patch**：修复bug，兼容老版本

**minor**：新增功能，兼容老版本

**major**：新的架构调整，不兼容老版本

------

### 版本格式

- version

  必须匹配某个版本

  如：1.1.2，表示必须依赖1.1.2版

- \> version

  必须大于某个版本

  如：>1.1.2，表示必须大于1.1.2版

  此外还有<=version、>=version、<version

- ～version

  大概匹配

  - ~1.1.2表示：1.1.2<= version <1.2.0
  - ~1.1表示: 1.1.0<= version <1.2.0
  - ~1表示: 1.0.0<= version < 2.0

- ^version

  兼容某个版本

- *和x

  - 1.2.x 表示：1.2.0 - 1.2.n
  - \* 表示： >=0.0.0的任意版本

- version1 - version2

  表示版本范围

  - 1.1.1 - 1.3.1 表示：1.1.1 - 1.3.1
